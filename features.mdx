## Create and Edit Pages

Follow this workflow to build and maintain your documentation.

<Steps>
  <Step title="Create a New Page" icon="plus">
    Navigate to your workspace and select `New Page`. Choose a template like "Guide" or "API Reference".
  </Step>
  <Step title="Edit Content" icon="edit-3">
    Use the visual editor for markdown and MDX. Add components like `<Callout>` or `<Steps>` directly.
  </Step>
  <Step title="Preview and Publish" icon="eye">
    Preview changes in real-time, then publish. Updates propagate instantly to your team.
  </Step>
</Steps>

## Collaborate Effectively

Сергій Москалюк supports multiple collaboration modes.

<Tabs>
  <Tab title="Real-time Editing" icon="activity">
    Multiple users edit simultaneously. Changes appear live with user avatars indicating activity.

    <Callout kind="tip">
      Assign roles like Editor or Viewer to control access.
    </Callout>
  </Tab>
  <Tab title="Version Control" icon="git-branch">
    Track changes with full history. Revert to previous versions or compare diffs.

    ```diff
    + Added new feature description
    - Fixed typo in workflow
      Collaborators: @user1, @user2
    ```
  </Tab>
</Tabs>

## Search and Organization Tools

Organize content hierarchically and search efficiently.

| Tool | Description | Use Case |
|------|-------------|----------|
| Global Search | Full-text search across all pages | Find "authentication" in any doc |
| Tags | Add labels like `api`, `guide` | Filter by topic |
| Folders | Nest pages in workspaces | Group by project phase |
| Breadcrumbs | Navigate page hierarchy | Quick backtracking |

Use tags for dynamic filtering: `tags={["feature", "guide"]}` in queries.

<Expandable title="Advanced Search Tips" default-open="false">
  Combine search with filters: "collaboration >10 results". Bookmark frequent queries for reuse.
</Expandable>

## Customize Your Space

Apply your brand, like the `#3B82F6` color, via configuration.

<CodeGroup tabs="YAML,JSON">
  ```yaml
  theme:
    primaryColor: "#3B82F6"
    logo: "/logo.svg"
  ```
  ```json
  {
    "theme": {
      "primaryColor": "#3B82F6",
      "logo": "/logo.svg"
    }
  }
  ```
</CodeGroup>

Update settings in your workspace dashboard. Changes apply site-wide without downtime.
